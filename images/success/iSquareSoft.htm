Role Security Engineer

Experience: 10+ years | Location: Bangalore

Key Responsibilities

  • As part of our engineering team, you will solve challenging problems to shape the future of automotive security and make products even more secure.
  • You will design, implement, and configure security features across multiple layers of Linux distribution: from hardware, bootloader, and kernel to end-user applications.
  • You integrate secure aspects of the Android platform and help application teams in developing a modern and secure customer experience.
  • Due to the high complexity of modern connected cars, you can apply your knowledge in hardware security, application security, cryptography, network security, and other security domains.

Cluster description

  • Linux based SW platform providing common automotive functions, like lifecycle, system functions, network, security, etc. to multiple Infotainment ECU projects/products.
  • Project is responsible for defining, developing and maintaining the platform, and integrating it into ECU Projects.
  • Automotive functions are developed in Center of Competencies (CoC), CoCs are end-to-end responsible to deliver end-customer functionality to a vehicle and to a customer.
  • The project is responsible also for developing and maintaining infrastructure to develop, build and test the platform. This includes CI system, build tools, test framework, test farm, and multiple other tools.
  • Currently platform is used in about 10 ECUs and the number is growing.

Must have requirements

  • Willingness to work in Bangalore on site.
  • Agile methodologies experience, e.g., Scrum or Kanban.
  • Comprehensive understanding of agile methodologies and extensive practical experience applying scrum patterns and techniques.
  • Hands-on experience in Linux or/and Android security (for example, Kernel hardening, Linux permissions, capabilities, namespaces, secure RPC).
  • Knowledge about isolation techniques and containerization mechanisms is a plus.
  • Thorough understanding of security principles, latest techniques, and operating system features (for example, secure boot, disk encryption, SELinux, integrity protection, cryptographic protocols).
  • Background with modern software development in one of the languages C++, Java, Kotlin. Experience with Python as a universal tool.
  • Experience with Continuous Integration and Delivery tools (for example, Jenkins, Zuul, Github Actions, or Gitlab CI).
  • Understanding of basic principles of threat modeling. Practical experience is a plus.

Soft Skills

  • Good skills and knowledge of servant leadership, facilitation, situational awareness, conflict resolution, continual improvement, empowerment, and increasing transparency.
  • Communicating with stakeholders comfortably and confidently.
